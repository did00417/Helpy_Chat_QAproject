# 🧪 AI 헬피챗 자동화 테스트 프로젝트

본 프로젝트는 **AI 헬피챗(AI Helpy Chat) 서비스**에 자동화 테스트를 적용하여
**QA 업무의 효율성과 제품 품질 향상**을 목표로 합니다.

**Selenium + Python + pytest** 기반으로 구성되어 있으며,
핵심 기능(로그인, 메시지 입력, AI 응답 검증 등)에 대한 자동화 시나리오를 포함합니다.

또한 본 테스트 코드는 **Jenkins**와 같은 **CI/CD** 환경에서도 실행 가능하도록 설계되어,
코드 변경 시 자동 테스트 수행, 테스트 결과 리포트 생성 등
지속적인 품질 관리를 위한 기반을 제공합니다.</br>

- 프로젝트 기간: 251208 ~ 251224
---

## 🔧 프로젝트 목표

- 테스트 효율성을 크게 향상합니다.
- 수동 테스트에서 놓치기 쉬운 추가 결함을 신속하게 발견할 수 있습니다.
- 서비스의 품질과 신뢰성을 지속적으로 개선합니다.
-  반복 작업을 줄여 업무 생산성을 극대화합니다.
- CI/CD 파이프라인과 연동되어 지속적인 통합 및 배포를 지원합니다.

## ⭐ 실행 방법
1. 환경 설정</br>
- 파이썬 버전 확인
    `python --version //3.11.9`
- 가상환경 활성화
    ` vsCode -> Select Interpreter ->  파이썬 버전 설정 후 venv 설정`

2. 의존성 설치(프로젝트에 필요한 패키지 설치)
- `pip install -r requirements.txt`

3. 개인 테스트 브랜치 생성
- `git checkout -b feature/name`

## 팀원 분들을 위한 깃 기본 가이드

규칙 -> master 브랜치말고 개인 브랜치를 사용하여 명령어 사용해주세요!

## ✅ 1. 변경된 파일 확인

작업한 뒤 반드시 변경 내용을 먼저 확인합니다.

`git status`

빨간색: 스테이지에 올라가지 않은 변경 파일
초록색: 커밋 준비 완료 상태(files staged)

## ✅ 2. 변경 파일 스테이지에 올리기 (add)
특정 파일만 올릴 때

`git add 파일명`

예시:

`git add tests/test_chat_ai.py`

전체 변경 파일 올릴 때 (주의 필요)
`git add .`


>❗ 팀 프로젝트에서는 가급적 git add . 은 조심해서 사용
>불필요한 파일까지 올라갈 수 있으므로 커밋 전에 git status 꼭 확인

## ✅ 3. 커밋 메시지 작성하기 (commit)

반드시 -m 옵션을 사용합니다.

`git commit -m "커밋 메시지 내용"`

예시:

`git commit -m "TC-CB-001, TC-CB-008 테스트 케이스 작성 완료"`

## ✅ 4. 원격 저장소로 올리기 (push)

현재 브랜치가 원격에 연결되어 있다면:

`git push`


처음 생성한 브랜치라 upstream 설정이 필요하면:

`git push -u origin 브랜치명`


예:

`git push -u origin feature/jungeun`
